# reduce

归约的解法：利用数学上的结合律，将线性执行转化为树状执行。
- 第一步：$N$ 个线程同时将数据两两配对相加，得到 $N/2$ 个结果。
- 第二步：$N/2$ 个线程继续配对相加，直到最后剩下一个值。
- 这样将时间复杂度从 $O(N)$ 降到了 $O(\log N)$。


## 总结
1. 初始版本（reduce_v0）中，我们采用了线程块内的归约策略，但由于线程闲置和线程束分化，资源利用率较低；
2. 通过引入预归约机制（reduce_v1），我们让每个线程处理两个全局内存元素，提高线程利用率；
3. 随后（reduce_v2），我们采用 **交错寻址**， tid < s 的判断方式替代取模运算，显著减少线程束内部的控制分歧并避免 bank conflict；
4. 最后，我们进一步优化了 warp 内部归约阶段，使用 **warp-level 原语** __shfl_down_sync 实现高效的 warp 内归约（reduce_v3），并通过 blockReduce 模式抽象出通用的线程块内归约结构，大幅提升指令级并行性和内存访问效率。
